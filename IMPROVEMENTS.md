# 改善レポート & 機能提案

未実装の項目のみを残し、通し番号を振り直したもの。
(実施済みの項目はコミット履歴を参照)

## A. コード面の残課題

1. **`npm run typecheck` が壊れている** — vue-tsc 1.8 が TypeScript 5.x と非互換でクラッシュする。`vue-tsc` を 2.x に更新すれば直る見込み。現状の検証手段は `npm run build`(vite/esbuild)のみで、型レベルの退行を検出できない。
2. **PresetPad / FacesPad のドラッグ処理が重複** — `elementFromPoint` + `data-*` 方式のドラッグ選択ロジックがほぼ同一実装で2箇所にある。composable(`usePadDrag`)に括り出せる。FAB ボタンの CSS(diagram-fab / pad-fab)も同様に共通化できる。
3. **半透明ソートの毎フレームアロケーション** — order 配列・ソート済み Float32Array を毎フレーム新規確保している。三角形数が小さいので実害はないが、再利用バッファ化すれば GC 圧をゼロにできる。

## B. UI / 機能面の改善点

4. **ダイヤグラム編集の Undo** — ドラッグ塗りで失敗したとき一発で戻せない。直近の選択状態スタックを持つだけで実装は軽い。
5. **セルのツールチップ** — ダイヤグラムのセルにホバー(長押し)で Du Val 領域番号・タイプ・軌道サイズを表示すると教材性が上がる。
6. **ズーム中の1本指パン** — ダイヤグラムはピンチでズーム/パンできるが、ズーム状態での1本指操作は塗りに割り当てられているため、片手でのパン手段がない。「ズーム中はふち部分のドラッグをパンにする」「パン/編集モードトグル」などの工夫の余地あり。
7. **Explode スライダーのスケール** — max=1000% の線形スライダーは前半に変化が集中する。対数スケール化するか、よく使う 0〜100% を広く取ると操作感が上がる。
8. **プリセット名の常時表示位置** — モバイルではパッドを閉じていると現在のプリセット名が見えない。3Dビュー上部に小さく名前をオーバーレイ表示しても良い。

## C. 追加機能の提案(面白そうなもの)

9. **殻の積み上げアニメーション** — プリセット適用時に、内側のセル層から順にフェードイン/落下合体させる。星型がどう「層」でできているかが直感的に伝わる(explode の逆再生としても実装できる)。
10. **スライドショー / モーフィング巡回** — 1→59 を自動で順送りするモード。ドラッグ送りで好評だった「形の遷移を眺める」体験の自動版。再生速度スライダー付き。
11. **3D→2D の逆引きハイライト** — 3D のファセットをクリックすると、ダイヤグラム側の対応セル(タイプ)が点滅・強調される。Focus モードの逆方向で、両画面の対応理解が完成する。
12. **STL / OBJ エクスポート** — 表示中の星型を 3D プリント用にダウンロード。メッシュは既に三角形リストなので変換は単純(法線向きと多様体性の検証は必要)。
13. **2D ダイヤグラムの SVG エクスポート** — 選択状態を反映した星型化図をベクターで保存。Wikipedia 風の図版を自作できる。
14. **ジャイロ回転** — スマホの傾き(DeviceOrientation)でモデルを回す。手の中で結晶を転がす感覚になり、モバイル体験が一段上がる。
15. **カスタム星型のギャラリー/共有** — 状態の URL エンコードができているので、自作の選択パターンに名前を付けて一覧化する「マイ星型集」も小さな拡張で作れる。
